# 队友接入说明

## 当前统一口径

- 为兼容现有接口，继续使用字段 `score`，但含义统一为**风险指数**：0 最低，100 最高。
- `risk_level` 与风险指数方向一致：`low` / `medium` / `high`。
- `behavior_profile.percentage` 是当前文本中的信号强度，不是科学概率，也不是对人物的评分。
- `behavior_profile.type` 使用具体行为名称，不使用“焦虑型”“控制型人格”等人物类型。

## 队友 B：AI 与接口

1. 将 [prompts.md](./prompts.md) 中 Prompt 1 用于 `/api/analyze`。
2. 将 Prompt 2 用于 `/api/chat`。
3. 保持现有 JSON 字段不变，避免影响前端联调。
4. 启用 JSON 输出并在服务端校验字段；失败时返回统一错误，不返回模型原始内容。
5. 用 [demo-cases.md](./demo-cases.md) 的 6 个案例检查误判、安全提醒和 JSON 格式。

## 队友 A：页面展示

1. 在分数旁明确显示“风险指数”，进度条越长表示风险越高。
2. 建议颜色：0–29 绿色，30–59 橙色，60–79 深橙或红色，80–100 红色。
3. 展示接口中的 `summary`，其中会包含总体判断和必要的不确定性。
4. “行为模式”建议改名为“检测到的行为信号”。
5. 在强度条下增加小字：“仅根据当前文本估算，不代表科学概率或人物诊断。”
6. `flags` 为空时显示“当前片段未发现明显风险信号”，不要留下空白。
7. “离场”可改为“暂停对话”；高风险时再显示“安全退出”。
8. 底部入口建议改为：“想继续梳理？我陪你分析下一句 →”。

## 推荐展示文案

```text
风险指数 72/100
需要警惕
仅根据当前聊天片段估算，不代表对人物的诊断。
```

```text
检测到的行为信号
信号强度仅用于比较当前文本中的表现，不代表科学概率。
```
