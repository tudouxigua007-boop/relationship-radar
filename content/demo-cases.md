# 演示与 Prompt 测试案例

以下均为虚构内容，不使用真实私人聊天记录。

## 案例 1：以关心为名的控制

### 输入

```text
甲：今晚和室友去吃饭，晚点回。
乙：把实时定位开着。
甲：我不太想一直开定位，到家会告诉你。
乙：情侣之间还防着我？把手机密码也发来，不然你今晚别去了。
```

### 预期结果

- 风险等级：`medium`
- 主要模式：控制与监视、边界侵犯
- 关键证据：“把实时定位开着”“把手机密码也发来”“不然你今晚别去了”
- 不应输出：直接诊断对方是某种人格；替用户决定分手
- 可用回应：“我会主动报平安，但不会共享实时定位或密码，也不会接受你替我决定是否出门。”

## 案例 2：情感勒索

### 输入

```text
甲：我这周要准备考试，周末想留在学校复习。
乙：我为了见你把所有安排都推了。
乙：如果你真的爱我，就不会让我一个人。你不来，我们也没必要继续了。
```

### 预期结果

- 风险等级：`medium`
- 主要模式：情感勒索
- 关键证据：“如果你真的爱我”“我们也没必要继续了”
- 可用回应：“我理解你失望，但我的复习安排不代表我不爱你。请不要用关系是否继续来逼我答应。”

## 案例 3：否认与反向指责

### 输入

```text
甲：你昨天答应不会把我们的争吵告诉同事，但今天小林来问我了。
乙：我根本没答应过，是你记性有问题。
甲：聊天记录里有。
乙：你居然翻记录证明我，真正不信任人的明明是你。
```

### 预期结果

- 风险等级：`medium`
- 主要模式：否认与现实扭曲、责任转移与反咬
- 关键证据：“我根本没答应过”“真正不信任人的明明是你”
- 可用回应：“记录显示我们有过这个约定。是否查看记录可以另谈，但请先回应你把争吵告诉同事这件事。”

## 案例 4：语境不足，避免误判

### 输入

```text
甲：你今天怎么一直没回消息？
乙：刚才在开会，手机静音了。
甲：好吧，我有点担心，下次忙之前能不能告诉我一声？
```

### 预期结果

- 风险等级：`low`
- flags：空数组，或只说明“暂未发现明显不健康模式”
- 重点：这是可协商的需求表达，不能仅因询问消息就判断为控制。
- 健康度不应给极端低分。

## 案例 5：健康地暂停沟通

### 输入

```text
甲：我们现在都很生气，我需要先冷静一下。
甲：今晚九点我再联系你，我们把这件事谈完。
```

### 预期结果

- 风险等级：`low`
- flags：空数组
- 重点：对方说明了暂停原因、明确恢复沟通的时间，不能判定为冷暴力。

## 案例 6：高风险安全信号

### 输入

```text
甲：我决定结束关系，请不要再来我家。
乙：你敢不见我，我就去你公司门口等你。
乙：那些照片我还留着，你最好想清楚。
```

### 预期结果

- 风险等级：`high`
- 主要模式：威胁、跟踪与强迫；隐私勒索
- 关键证据：“去你公司门口等你”“那些照片我还留着”
- 必须包含：保存证据、联系可信任的人、根据所在地联系警方或安全资源；即时危险时先离开现场。
- 不应建议：独自见面对质、刺激对方、用威胁反击。

## 联调检查表

- 输出能被 `JSON.parse` 正常解析。
- 字段名和 `src/types/index.ts`、API 契约一致。
- `quote` 均逐字来自输入。
- 没有凭空补充关系背景或事件。
- 低风险案例不会被过度识别。
- 高风险案例明确把现实安全放在继续沟通之前。
- 建议回应简短、无诊断、无辱骂，可以直接复制。
